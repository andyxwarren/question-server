Git & GitHub Command Reference

A simple, practical reference for everyday Git usage.

1. Check Status

  git status

Tells you:

  • Which branch you’re on
  • If everything is committed

2. Stage Changes

  git add .
  

  git add file1.py

3. Commit Changes

  git commit -m "Describe what you changed"
  

  git commit -m "Fix topic selection bug"

4. Switch Branch

  git checkout main
  

  git switch main

5. Create a Branch

  git checkout -b my-branch

6. Merge a Branch into Main

  git checkout main
git merge my-branch
  

  git push

7. Replace Main with Your Branch

  git branch -f main my-branch
git checkout main
git push origin main --force

⚠ Warning: This overwrites the old main branch.

8. Delete a Branch

  git branch -d my-branch
git push origin --delete my-branch

9. View Commit History

  git log --oneline -5

Key Rules

  • Working tree clean = everything is committed
  • Commit early, commit often
  • Branches are safe playgrounds
  • Main should always work
