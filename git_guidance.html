How to Upload Your Changes from Windsurf to GitHub

This page shows the exact commands you need, in the correct order. You will use these over and over again.

Step 1 — Check what you’ve changed

Always start here.

  git status

You are looking for:

  • modified: files → files you edited
  • untracked files: → new files you created
  • working tree clean → nothing to upload

Step 2 — Stage your changes

This tells Git what you want to include in the upload.

  git add .

Think: “Put everything I’ve changed into the basket.”

Step 3 — Commit (save a snapshot)

This saves your changes locally. This does NOT upload yet.

  git commit -m "Describe what I changed"

Step 4 — Push to GitHub (upload)

This is the step people forget.

  git push

This sends your commits to GitHub and updates GitHub Pages.

The Full Windsurf Workflow (memorise this)

  git status
git add .
git commit -m "update"
git push

You will do this sequence every time you change files.

Quick Checks if Things Don’t Update

  • Did you run git commit?
  • Did you run git push?
  • Does git status say working tree clean?
  • Are you on main?
  git branch

Emergency One-Line Upload

Fine for solo projects when you don’t care about messages.

  git add . && git commit -m "update" && git push
Rule of thumb:
If GitHub hasn’t changed, you probably forgot git push.
